# Excel2TeX MVP構成および環境構築手順

本ドキュメントは `specification.md` の内容に基づき、最速で動作を確認するための MVP（Minimum Viable Product：実用最小限の製品）の構成と、その環境構築手順を定義する。

## 1. MVP（実用最小限の製品）の構成

仕様書のすべての機能を最初から実装するのではなく、「アプリとして起動し、ファイルからTeXコードを生成してコピーできる」というコアバリューを体験できる状態を最初の目標（MVP）とする。

### 1.1 MVPで実装する機能

1. **ファイル入力機能 (UC-01)**
   - まずは実装が容易な `.csv` の読み込みに対応する（余裕があれば `.xlsx` も追加）。
2. **変換・プレビュー機能 (UC-03)**
   - 読み込んだデータを元に、プレーンな `table` / `tabular` 環境のコードを生成する。
   - 生成されたTeXコードを右ペインに表示する。
   - *※キャプション、ラベル、罫線などの「付加情報・スタイル設定(UC-02)」はMVP動作確認後の次のステップで実装する。*
3. **コピー機能 (UC-04)**
   - ボタン一つでプレビューされたTeXコードをクリップボードにコピーする。

### 1.2 ファイル構成（MVP段階）

仕様書の「関心の分離」に基づき、UIとロジックを分割した以下のシンプルな構成とする。

```text
excel2tex/
├── main.py              # UIロジック (Fletでの画面描画・イベント処理)
├── converter.py         # 変換ロジック (pandasを使ったTeX変換処理)
├── requirements.txt     # 必要なパッケージ一覧
└── tests/
    └── test_converter.py # 変換ロジックの単体テスト
```

## 2. 環境構築の手順

Pythonの標準機能である `venv`（仮想環境）を利用して、システム環境を汚さずに開発環境を構築する。

### 2.1 仮想環境の作成と有効化

プロジェクトのルートディレクトリ（`excel2tex/`）で以下のコマンドを実行し、仮想環境を作成・有効化する。

```bash
# 仮想環境「.venv」を作成
python3 -m venv .venv

# 仮想環境を有効化 (Linux / macOSの場合)
source .venv/bin/activate

# ※Windows (コマンドプロンプト/PowerShell) の場合
# .venv\Scripts\activate
```

### 2.2 パッケージのインストール

開発に必要なライブラリを記載した `requirements.txt` を用意し、インストールを行う。

**`requirements.txt` の内容:**
```text
flet          # UIフレームワーク
pandas        # データ処理
openpyxl      # Excel(.xlsx)読み込み用（pandasの裏側で必要）
pytest        # テストフレームワーク
```

**インストールコマンド:**
```bash
pip install -r requirements.txt
```

### 2.3 動作確認（テスト実行）

仮想環境とライブラリが正しくセットアップされているか確認するため、テストを実行する。
（※テストファイル作成後に実施）

```bash
pytest tests/
```
