# Excel2TeX システム仕様書

## 1. システム概要
「Excel2TeX」は、理系の学生や研究者が論文・レポート執筆時に直面する「ExcelやCSVで作成した表データをLaTeX（TeX）のコードに変換する」という作業負担を軽減するためのデスクトップ・Webアプリケーションである。ユーザーが直感的に操作できるGUIを提供し、設定の変更をリアルタイムでTeXコードのプレビューに反映させることを目的とする。

## 2. 要求定義

### 2.1 ターゲットユーザー
- 日常的に実験レポートや論文を作成する理系の学生・研究者
- TeXの表作成（`\begin{tabular}`など）に苦手意識がある、または作業時間を短縮したい人

### 2.2 機能要件 (WHAT)
1. **ファイル入力機能**: `.xlsx` および `.csv` 形式のファイルを読み込むことができる。
2. **TeXコード変換・出力機能**: 読み込んだデータを元に、`table` 環境および `tabular` 環境で囲まれたTeXコードを生成する。
3. **付加情報設定機能**: 
   - キャプション（`\caption{}`）の設定
   - ラベル（`\label{}`）の設定
4. **スタイル・装飾設定機能**:
   - 罫線の有無、または `booktabs` パッケージを用いたスタイル指定
   - セルの配置指定（中央 `c`、左 `l`、右 `r`）
   - ※将来拡張：セルの結合（`\multicolumn`）対応
5. **リアルタイムプレビュー機能**: データをアップロードした後、左側の設定パネルを変更すると、右側のコード出力エリアに即座に変更が反映される。
6. **クリップボードへのコピー機能**: ボタン一つで生成されたコードをクリップボードにコピーできる。

### 2.3 非機能要件 (HOW)
- **プラットフォーム**: Windows / macOS / Linux で動作するデスクトップアプリケーションを基本とし、同一コードベースでWebブラウザからの利用にも対応させる。
- **情報デザイン（使いやすさ）**: 1つの画面で完結するシンプルな操作手順とする。「ゲシュタルトの法則（近接の法則・類同の法則）」に則り、関連する設定項目を視覚的にグループ化する。
- **品質と保守性**: UI（画面表示）とロジック（変換処理）を分離し、`pytest` を用いた自動テスト（単体テスト）を容易に実行できるアーキテクチャとする。

## 3. ユースケース

### 3.1 アクター
- **ユーザー**: 表データをTeXコードに変換したい学生・研究者

### 3.2 ユースケース図（構成要素）
- UC-01: ファイルをアップロードする
- UC-02: 表のレイアウトや装飾を設定する
- UC-03: TeXコードのプレビューを確認する
- UC-04: TeXコードをコピーする

### 3.3 ユースケース記述（メインシナリオ: 表の変換とコピー）
1. **事前条件**: ユーザーのPC内に変換したい `.xlsx` または `.csv` ファイルが存在する。
2. **基本フロー**:
   1. ユーザーがアプリを起動する。
   2. ユーザーが指定のエリアにファイルをドラッグ＆ドロップ（またはファイル選択ダイアログから選択）する。(UC-01)
   3. システムがファイルを読み込み、デフォルト設定で変換されたTeXコードを右側のプレビューエリアに表示する。(UC-03)
   4. ユーザーが左側の設定エリアで「キャプション」と「ラベル」のテキストを入力する。(UC-02)
   5. システムが即座に入力内容を反映し、プレビューエリアのコードを更新する。(UC-03)
   6. ユーザーが「コピー」ボタンをクリックする。
   7. システムがTeXコードをクリップボードに保存し、「コピーしました」というフィードバックを提示する。(UC-04)
3. **事後条件**: ユーザーのクリップボードに目的のTeXコードが格納されている。

## 4. 外部設計（インタフェース設計）

### 4.1 画面構成（左右分割レイアウト）
ユーザーの自然なメンタルモデル（左で操作し、右で結果を確認する）に合わせ、画面を左右に分割する。

* **【左ペイン】設定エリア (入力・原因)**
  - ファイルアップロードゾーン
  - **付加情報パネル**：キャプション入力欄、ラベル入力欄
  - **スタイルパネル**：罫線・配置・booktabs使用などのチェックボックス/ラジオボタン
  *(※各パネル間に適度な余白を設け、近接の法則を適用)*

* **【右ペイン】プレビュー＆出力エリア (出力・結果)**
  - 生成されたTeXコードを表示する複数行のテキストエリア（読み取り専用）
  - 右下または目立つ位置に「コピーする」ボタンを配置。アフォーダンス（押せそう感）を強調したデザイン・色使いとする。

## 5. 内部設計（アーキテクチャと技術選定）

### 5.1 使用技術
- **プログラミング言語**: Python 3.x
- **UIフレームワーク**: Flet (UIの構築およびクロスプラットフォーム対応)
- **データ処理ライブラリ**: pandas (Excel/CSVの読み込みおよびデータフレーム操作)
- **テストフレームワーク**: pytest (変換ロジックの単体テスト)

### 5.2 モジュール構成（関心の分離）
システムは大きく以下の2つのモジュールに分けて実装する。
1. **`converter.py` (変換ロジック)**:
   - Fletに依存しない純粋なPython関数。
   - `pandas.DataFrame` と各種設定（キャプション文字列、罫線フラグなど）を受け取り、TeX形式の文字列（`str`）を返す。
2. **`main.py` (UIロジック)**:
   - Fletを用いた画面描画とイベント処理を行う。
   - ユーザーの入力を検知して `converter.py` の関数を呼び出し、結果を画面に反映させる。

### 5.3 テスト計画
第8回講義「実装・テスト・運用・保守」に基づき、品質確保のための単体テストケースを整備する。`pytest` を用いて、画面を立ち上げずに自動実行する。

**テストケース例 (`test_converter.py`):**
| テストID | テスト対象・入力データ | 期待される出力（Expected Result） |
|---|---|---|
| TC-001 | 2行2列の正常なCSVデータ<br>設定: デフォルト | `\begin{table}`、`\begin{tabular}{cc}`、データ行が `&` と `\\` で結合されて含まれていること。 |
| TC-002 | 任意のCSVデータ<br>設定: caption="実験1", label="tab:exp1" | コード内に `\caption{実験1}` および `\label{tab:exp1}` が正確に出力されていること。 |
| TC-003 | データが空のCSVファイル | 空であることを示す適切なエラーハンドリング、またはヘッダーのみのコードが出力されること。 |
