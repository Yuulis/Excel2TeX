# inside-docs ガイド

`inside-docs` は、Excel2TeX の仕様、実装計画、フェーズごとの実装報告、
および現在の実装状況を保管する開発者向けドキュメント領域です。

## 最初に読む資料

- [現在の実装状況](project-status/implementation-status-2026-07-17.md)
  - 2026-07-17 時点の実装済み機能、構成、検証結果、既知の制約をまとめた現況資料です。
- [システム仕様書](specification.md)
  - プロジェクト全体の目的、要求、画面・内部設計を記録した基礎仕様です。

## ディレクトリ構成

| パス | 位置づけ |
|---|---|
| `project-status/` | 現在の実装状況を日付付きで保存するスナップショット |
| `mvp-implementation/` | MVP の計画と初期実装報告 |
| `additional-info-feature/` | キャプション・ラベル機能の実装報告 |
| `extended-features/` | 拡張仕様、ロードマップ、拡張機能の実装報告 |
| `table-editor-feature/` | インタラクティブなテーブル編集機能の実装報告 |
| `specification.md` | 当初のシステム仕様書 |

## 読み方に関する注意

各 `report.md` と計画書は、その機能を実装した時点の履歴資料です。テスト件数、
ファイル配置、残課題などは現在と異なる場合があります。現行コードの全体像は、
`project-status/` 内の最新日付の資料を正として参照してください。

過去資料は意思決定と実装経緯を残すため、内容を現在形へ書き換えず保存しています。
